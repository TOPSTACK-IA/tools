Manual de Assets & Caminhos

Guia técnico para deploy seguro em GitHub Pages e Netlify

1. A Regra de Ouro (Linux vs Windows)
Case Sensitivity (Maiúsculas e Minúsculas)

O Windows ignora diferenças entre maiúsculas e minúsculas, mas os servidores (Netlify/GitHub) usam Linux, que diferencia tudo.


  • ❌ Erro Comum: Arquivo chama Logo.png e no código está logo.png.
  • ✅ Solução: Sempre use nomes de arquivo em minúsculo e sem espaços.
2. Lógica de Navegação (./ vs ../)
Para arquivos na Raiz (Ex: index.html)

Use ./ para dizer "nesta pasta".

<img src="./assets/logo.png">
<link href="./styles.css">
Para arquivos em Pastas (Ex: blog/index.html)

Use ../ para dizer "volte uma pasta para trás" e achar a raiz.

<!-- Saindo da pasta blog para achar a pasta assets -->
<img src="../assets/logo.png">
<link href="../styles.css">
